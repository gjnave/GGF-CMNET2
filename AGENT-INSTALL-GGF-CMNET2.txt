GGF CMNET2 COLORIZATION STUDIO
AI AGENT INSTALLATION HANDOFF
Style: Get Going Fast Agentic Install v2

Purpose
-------
This file is for a user who gives one file to an AI agent and says:

    Install this app for me.

The agent is not building an installer product.
The agent is installing GGF CMNET2 Colorization Studio on the user's machine
and leaving them with a clean way to run it.

GGF CMNET2 is a local browser app for CMNET2 / ColorMNet. It colorizes
black-and-white images and videos using color reference images or reference
frames. It runs locally with CUDA PyTorch on NVIDIA GPUs.

Before Major Changes
--------------------
Briefly explain major changes before doing them:

- The app will be cloned from GitHub.
- A local Python virtual environment will be created inside the app folder.
- CUDA PyTorch and the app's Python dependencies will be installed into that
  virtual environment only.
- Model files will be downloaded to the app folder on the selected drive.
- A click-to-run BAT file and START-HERE text file will be created.

Approximate model download size:

    About 700 MB total for the required CMNET2 release files.


Install Steps
-------------
Replace C: with the drive chosen by the user.

1. Create the GGF app folder:

        if not exist C:\GGF mkdir C:\GGF
        if not exist C:\GGF\GGF-CMNET2 mkdir C:\GGF\GGF-CMNET2

2. Clone or update the app source:

        if exist C:\GGF\GGF-CMNET2\GGF-CMNET2\.git git -C C:\GGF\GGF-CMNET2\GGF-CMNET2 pull --ff-only
        if not exist C:\GGF\GGF-CMNET2\GGF-CMNET2\.git git clone https://github.com/gjnave/GGF-CMNET2.git C:\GGF\GGF-CMNET2\GGF-CMNET2

   If `git pull --ff-only` fails because the folder has local changes, stop and
   report the issue. Do not overwrite user changes.

3. Create or reuse the app-local virtual environment:

        cd /d C:\GGF\GGF-CMNET2\GGF-CMNET2
        if not exist .venv\Scripts\python.exe py -3.10 -m venv .venv

4. Upgrade pip inside the venv:

        .venv\Scripts\python.exe -m pip install --upgrade pip

5. Install CUDA PyTorch inside the venv:

        .venv\Scripts\python.exe -m pip install --index-url https://download.pytorch.org/whl/cu128 torch torchvision torchaudio

6. Install app dependencies inside the venv:

        .venv\Scripts\python.exe -m pip install opencv-python pillow scikit-image tqdm numpy einops tensorboard gradio

7. Create model folders:

        if not exist weights mkdir weights
        if not exist models mkdir models
        if not exist models\checkpoints mkdir models\checkpoints

8. Download required model files with resumable curl.

   Use curl.exe. Existing complete files can be reused. If a file is partial,
   rerun the same command; `-C -` resumes the download.

   CMNET2 ColorMNet weights:

        curl.exe -L --fail --retry 12 --retry-delay 5 --retry-all-errors -C - -o weights\DINOv2FeatureV6_LocalAtten_s2_154000.pth https://github.com/dan64/cmnet2/releases/download/v1.0.0/DINOv2FeatureV6_LocalAtten_s2_154000.pth

        Expected size: 494884817 bytes

   DINOv2 ViT-S/14 weights:

        curl.exe -L --fail --retry 12 --retry-delay 5 --retry-all-errors -C - -o models\checkpoints\dinov2_vits14_pretrain.pth https://github.com/dan64/cmnet2/releases/download/v1.0.0/dinov2_vits14_pretrain.pth

        Expected size: 88283115 bytes

   ResNet18 weights:

        curl.exe -L --fail --retry 12 --retry-delay 5 --retry-all-errors -C - -o models\checkpoints\resnet18-5c106cde.pth https://github.com/dan64/cmnet2/releases/download/v1.0.0/resnet18-5c106cde.pth

        Expected size: 46827520 bytes

   ResNet50 weights:

        curl.exe -L --fail --retry 12 --retry-delay 5 --retry-all-errors -C - -o models\checkpoints\resnet50-19c8e357.pth https://github.com/dan64/cmnet2/releases/download/v1.0.0/resnet50-19c8e357.pth

        Expected size: 102502400 bytes

   DINOv2 source zip:

        curl.exe -L --fail --retry 12 --retry-delay 5 --retry-all-errors -C - -o models\facebookresearch_dinov2_main.zip https://github.com/dan64/cmnet2/releases/download/v1.0.0/facebookresearch_dinov2_main.zip

        Expected size: 1226660 bytes

9. Verify model file sizes:

        powershell -NoProfile -ExecutionPolicy Bypass -Command "Get-Item 'weights\DINOv2FeatureV6_LocalAtten_s2_154000.pth','models\checkpoints\dinov2_vits14_pretrain.pth','models\checkpoints\resnet18-5c106cde.pth','models\checkpoints\resnet50-19c8e357.pth','models\facebookresearch_dinov2_main.zip' | Select-Object FullName,Length"

   Confirm:

        weights\DINOv2FeatureV6_LocalAtten_s2_154000.pth    494884817
        models\checkpoints\dinov2_vits14_pretrain.pth       88283115
        models\checkpoints\resnet18-5c106cde.pth            46827520
        models\checkpoints\resnet50-19c8e357.pth            102502400
        models\facebookresearch_dinov2_main.zip             1226660

   If any file is smaller than expected, rerun that file's curl command.
   If any file is larger than expected, delete that file and download it again.

10. Extract DINOv2 source:

        powershell -NoProfile -ExecutionPolicy Bypass -Command "Expand-Archive -LiteralPath 'models\facebookresearch_dinov2_main.zip' -DestinationPath 'models' -Force"

    Confirm this folder exists:

        models\facebookresearch_dinov2_main

11. Create the click-to-run file:

    File:

        C:\GGF\GGF-CMNET2\CLICK-ME-TO-RUN.bat

    Exact content:

        @echo off
        setlocal
        cd /d "%~dp0GGF-CMNET2"
        call "%~dp0GGF-CMNET2\run.bat"

Troubleshooting Method
----------------------
Use this loop:

1. Attempt
2. Observe
3. Read the exact error
4. Diagnose the likely cause
5. Apply one targeted fix
6. Re-test

Do not do shotgun reinstalls.
Do not repeatedly run the same failing command without changing anything.
Do not overwrite local user changes.

Common issues:

- Missing Git, Python, curl, or CUDA/NVIDIA support:
  Stop and send the user to https://setup.getgoingfast.pro

- Model download fails:
  Rerun the exact curl command. It should resume because of `-C -`.

- Model file has the wrong size:
  Smaller than expected means partial download; rerun curl.
  Larger than expected means corrupted/resume mismatch; delete and redownload.

- GUI starts but colorization fails:
  Check CUDA, model sizes, extracted DINOv2 source, and selected target/reference
  files.

- Port 7867 is busy:
  Run:

        cd /d C:\GGF\GGF-CMNET2\GGF-CMNET2
        .venv\Scripts\python.exe app\app.py --port 7868
